### Thread 1: Gateway Router
- Streaming and non-streaming response handling (SSE piping when `stream=true`)
- Request/response logging middleware capturing: `tenant_id`, model, tokens used, latency, response code
- **Load balancing:** Weighted round-robin across upstream providers (configurable per model per tenant)
- **Failover:** On upstream 500/503, retries next provider in weighted list; max 2 retries

### Thread 2: Rate Limiting & Quotas
- **Rate limit:** 100 RPM per tenant, stored in Redis keyed by `tenant_id`
- **Algorithm:** Sliding window with token bucket for bursting (20 burst tokens above base)
- **At limit:** Returns `429 Too Many Requests` with `Retry-After` header
- **Rate limit headers:** `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`
- **Quota system:** Monthly token quota tracked in Redis; prod tenant has 10M tokens/month
- **Pre-flight quota check** added to router; returns `429` with `X-Quota-Exceeded: true` when exhausted
