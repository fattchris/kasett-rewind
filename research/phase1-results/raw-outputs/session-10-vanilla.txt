### Six Development Threads

**Thread 1 — Plugin Architecture**
- Registers via `register(api: PluginAPI)` exported from `src/index.ts`
- OpenClaw plugin loader calls `register()` on startup; config in `openclaw.plugin.json`
- Three hooks via `api.on()`:
  - `before_compaction` — inject steering prompt
  - `after_compaction` — parse `[THREAD_META]` block and store
  - `context_load` — inject orientation from most recent meta

**Thread 2 — ThreadMeta Schema**
- Schema: `{main: string, sub: [string, string, string]}`
- JSONL event type/field name: `kaspiett` (legacy name from before project rename; kept for backwards compatibility)
- Storage path: `/home/node/.openclaw/agents/main/sessions/`
- Rolling window: `rolling_window_size=5` compactions

**Thread 3 — SessionReader**
- Reads JSONL files from `/home/node/.openclaw/agents/main/sessions/`
- JSONL event format: `{timestamp, type, data: {kaspiett: ThreadMeta, summary: string}}`
- Filters for `data.kaspiett != null`; returns last N events per `rolling_window_size=5`
- Error handling: `try/catch` around file reads; missing directory or corrupt JSONL returns empty array; plugin degrades gracefully to basic steering

**Thread 4 — Benchmark Harness**
- Model: `anthropic/claude-sonnet-4-6` via `https://openrouter.ai/api/v1/chat/completions`
- Settings: `temperature=0`, `max_tokens=4096`, 3-second delay between calls
- `phase1_fixture_count=10` synthetic sessions across tiers 1–3
- Tier 3 fixtures: 5+ threads, 8–12 key state values, 80–100 turns, greppable values (URLs, UUIDs, version strings, file paths)
- Fixtures written to `research/phase1-results/fixtures/`
- Metrics: **TRR** (Thread Retention Rate), **KSSR** (Key State Survival Rate), Cohen's d effect size

**Thread 5 — Steering Prompt Builder**
- `analyzeThreads()` classifies threads as `core/fresh/fading`
- `buildSteeringPrompt()` takes `ThreadAnalysis` + previous metas → structured prompt
- With no previous metas: returns basic format instructions only (no thread history section); LLM still guided to produce `[THREAD_META]` block

**Thread 6 — THREAD_META Parser**
- `parseCompactionOutput()` uses regex `/\[THREAD_META\]\s*\n([\s\S]*?)\n?\s*\[\/THREAD_META\]/`
- Parses `main:`, `sub1:`, `sub2:`, `sub3:` lines → returns `{summary: string, meta: ThreadMeta|null}`
- If block absent: returns `{summary: raw_output.trim(), meta: null}`; plugin falls back to basic steering on next compaction
- Test cases covered: happy path, missing sub3, no block, extra whitespace, nested brackets
