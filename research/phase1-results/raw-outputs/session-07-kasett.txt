### Thread 2: Embedding Model Cost vs. Accuracy Tradeoff
Resolved with an **adaptive embedding strategy**:
- Queries ≤100 chars → `text-embedding-3-small` (1536d, $0.02/1M tokens) → routed to small index
- Queries >100 chars → `text-embedding-3-large` (3072d, $0.13/1M tokens) → routed to primary index
- Both models live in production as of v1.2.0 deployment

### Thread 3: Memory Persistence Layer
Designed a three-tier storage architecture:
- **S3** for raw memory content (durable, cheap; 90-day TTL for working memory, indefinite for long-term)
- **pgvector** for vector embeddings (queryable)
- **PostgreSQL** for metadata (filterable)
- **Async write path**: memory_id returned immediately after S3 write; embedding generated via SQS queue + Lambda workers (1–5s lag)
- **Status endpoint** added: `GET /v1/memories/{id}/status` returns indexing state and embedding model info
- **Reindex script** added to vector.ts exports; full reindex ~12 min with 10 parallel Lambda workers (~$13 cost at 100K memories)
- **Backup strategy**: RDS automated snapshots daily; index rebuildable from S3
