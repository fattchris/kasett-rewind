# Conversation Summary

## Key Decisions
- **Auth Flow**: Using JWT instead of OAuth. OAuth would complicate SDK integration for partners, and third-party auth providers aren't needed in this phase. JWT keeps the API surface minimal with option to revisit later if needed.
- **Postgres Version**: Targeting Postgres 15.4 for the migration to get partitioning improvements without committing to version 16 yet (RDS support for 16 is still spotty in the region).

## Technical Configuration & Infrastructure
- **Staging Deployment URL**: https://api-staging.molt.ai (standard staging endpoint with certs already provisioned)
- **Gateway Config Location**: `/etc/molt/gateway.toml` - controls upstream routing, rate-limit policy, and JWT verifier settings
- **Production Rate Limit**: 250 requests per second per tenant (will be raised after observing real traffic patterns)
- **Gateway Restart Command**: `kubectl rollout restart deploy/gateway -n molt-prod` (rolling restart takes ~90 seconds with PDB)
