The session focused on coordinating multiple parallel workstreams for the Molt platform, with particular attention to auth infrastructure and deployment logistics. The team is working toward an August 1 launch date set by Walton, contingent on marketing budget availability. Several key technical decisions were made: the auth flow will use JWT rather than OAuth to simplify the SDK story for partners, and the postgres migration will target version 15.4 to get partitioning improvements without committing to the still-spotty RDS support for version 16 in their region.

The staging auth gateway was deployed to https://api-staging.molt.ai using existing cert provisioning. The gateway reads its config from /etc/molt/gateway.toml at startup, controlling upstream routing, rate-limit policy, and JWT verifier settings. The prod gateway is currently configured for 250 requests per second per tenant, with plans to raise limits after observing real traffic patterns. Rolling restarts after config changes use `kubectl rollout restart deploy/gateway -n molt-prod`, completing in about 90 seconds with their PodDisruptionBudget.
