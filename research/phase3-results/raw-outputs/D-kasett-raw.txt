The session covered status updates and logistics for the dashboard v1 release, which is locked for GA on June 30 (firm date due to July 2 investor demo). Daniel is leading the dashboard work after the reorg and has been moving it faster than the previous owner. The dashboard beta now has 14 active customers (added four last week). The team is running React 19.0.0-rc.1 to use the new use() hook for streaming data sections, with plans to downgrade to stable once 19.0 ships.

Multiple work streams are progressing toward end-of-week milestones: API rate limiting (Andrew, Walton, Jim), perf work (Walton, Daniel, Thomson, Anna), dashboard refresh (Jim, Daniel), auth refactor (Anna, Andrew, Walton), schema review (Anna, Jim), steering integration (Andrew, Jim), migration (Andrew, Anna, Daniel), and orientation prompt logic (Walton). Various minor dependencies (snapshot writer, validator, parser path, queue worker, status check, feedback loop, cross-session index) were flagged as potentially slipping but not blocking their dependent work.

Two key decisions emerged: data export was punted to Q3 (not in v1) despite three customer requests, because it complicates the auth model and the team doesn't want to delay v1. Analytics instrumentation is stalled waiting on Segment API access — Anna submitted the request two weeks ago but their onboarding is slow; the instrumentation code itself is ready. QA reported a bug where NaN displays when no data is present (aggregation function returns NaN and formatter doesn't guard); Daniel has a fix in PR but it hasn't merged yet.
