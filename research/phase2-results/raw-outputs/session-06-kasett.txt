We implemented end-to-end encryption for the agent message bus service. The core work centered on building an envelope encryption pattern using AWS KMS, with AES-256-GCM for message encryption and HMAC-SHA256 for authentication. The encryption module handles per-message data key generation from KMS, encrypts the payload, and stores the encrypted data key alongside the ciphertext.

Key rotation became a significant focus. We configured automatic rotation every 90 days for the KMS key, with application-level data key rotation. Old messages remain decryptable through KMS's versioned key management. We added rotation audit logging to CloudWatch with 90-day retention and built an admin endpoint to manually flush the data key cache when rotation occurs.

Performance testing revealed that raw KMS GenerateDataKey calls added ~40ms per message, dropping throughput from 12,400 to 3,200 messages/second. We solved this with data key caching: 300-second TTL, max 1000 cached keys per tenant. This recovered throughput to 11,800 messages/second with only 12ms p99 latency — a 5% overhead versus plaintext. The cipher suite uses AES-256-GCM for encryption, HMAC-SHA256 for authentication, and RSA-OAEP for asymmetric operations when needed.

We completed the security design documentation and promoted the release from v5.0.0-rc2 to v5.0.0, then deployed to production.
