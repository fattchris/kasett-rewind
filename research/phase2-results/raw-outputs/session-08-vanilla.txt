## Five Development Threads

### Thread 1: Orchestrator
- Request router built for incoming agent requests
- Routes to correct Kubernetes pods in molt-agents-prod namespace
- Sticky sessions implemented with Redis backing
- Calls K8s API to list available pods and select least-loaded
- Falls back to least-loaded pod on failure
