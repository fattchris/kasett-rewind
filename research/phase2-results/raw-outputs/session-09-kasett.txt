We conducted SOC 2 Type II audit preparation across five parallel workstreams. The audit logging pipeline was upgraded to version 2.2.0 with immutable storage — events ingested at https://audit.moltaicorp.com/v1/events/ingest flow through Kinesis and Lambda into S3 bucket arn:aws:s3:::molt-audit-logs-prod-us-east-1, encrypted with KMS key arn:aws:kms:us-east-1:843979154439:key/c3d4e5f6-a7b8-9012-c3d4-e5f6a7b89012. We enabled S3 Object Lock in COMPLIANCE mode with 365-day retention to ensure tamper-proof audit trails. The Terraform configuration at /home/node/.openclaw/workspace/repos/molt-infra/compliance/audit/pipeline.tf was successfully applied.

Pentest finding PENTEST-2026-042 (high severity SSRF vulnerability in the agent proxy service) is being remediated on schedule — we're on day 5 of the 14-day SLA. The fix includes URL allowlist validation, blocking of internal IP ranges (10.0.0.0/8, 169.254.0.0/16, 172.16.0.0/12), and request signing for trusted endpoints. The code is in PR review.

IAM policy remediation moved from v8.3 to v8.4, addressing three categories of over-permissions: replacing s3:* with specific actions (s3:GetObject, s3:PutObject, s3:ListBucket) scoped to the audit bucket, removing unnecessary ec2:Describe* from Lambda functions, and adding conditional constraints to iam:PassRole (limited to lambda.amazonaws.com and ecs-tasks.amazonaws.com). These changes enforce least-privilege access per SOC2-CC6.1-CC7.2.

Vendor security assessment VSA-2026-017 for OpenAI is in progress — we documented the data processing agreement terms, confirmed no training on customer data, verified GDPR compliance, and compiled the subprocessor list (AWS, Stripe, OpenAI, Twilio). OpenAI received a Medium risk rating due to LLM prompt processing, though they maintain SOC 2 Type II certification and the DPA prohibits persistent customer data storage. The DPA is pending signature.

The SOC 2 evidence tracker was updated to reflect completed work (audit pipeline immutability satisfies CC6.1-CC7.2), in-progress items (PENTEST-2026-042 remediation, IAM v8.4 deployment pending), and blocked items (VSA-2026-017 DPA signature).
