We responded to a production auth outage caused by a JWT validation bug in v3.1.0. Users were getting 401 errors when trying to refresh tokens at the auth endpoint. The root cause was a milliseconds/seconds unit mismatch in the token expiry check — the code was comparing `Date.now()` (milliseconds) directly against the JWT `exp` claim (seconds), causing all tokens to appear expired immediately.

The bug was isolated to the JWT middleware file, and we confirmed 340 users were affected over approximately 70 minutes. We built and deployed a hotfix (v3.1.0-hotfix) that corrected the comparison by dividing `Date.now()` by 1000. We validated the fix using a specific affected session ID, confirmed the endpoint returned to baseline error rates, and verified that users would naturally re-authenticate without manual intervention.

After resolving the immediate issue, we added preventive measures: a new unit test specifically guarding against milliseconds/seconds regressions, and an ESLint rule to catch this pattern statically. We documented the incident in a postmortem covering timeline, impact, root cause, fix, and prevention, then archived the incident.
