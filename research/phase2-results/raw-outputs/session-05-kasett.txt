We built multi-region Terraform IaC for ECS Fargate agent deployment across us-east-1 and us-west-2. The module at `/home/node/.openclaw/workspace/repos/molt-infra/terraform/modules/agent-cluster/main.tf` handles VPC setup, ECS cluster provisioning, Fargate task definitions, and auto-scaling configuration. VPC CIDRs are non-overlapping (10.0.0.0/16 for us-east-1, 10.1.0.0/16 for us-west-2) with VPC peering established between them.

We worked through DNS resolution across VPCs using Route 53 private hosted zones associated with both VPCs, allowing agents in us-west-2 to resolve the us-east-1 health endpoint. The ECS task definition configures 1 vCPU and 2GB RAM per container by default, with environment variables pulled from SSM Parameter Store.

Cost optimization was a major focus. We evaluated three patterns: always-on ($110/month for 10 idle tasks), scale-to-zero ($0 idle cost but 30-second cold starts), and warm standby ($22/month for 2 tasks with no cold starts). We chose warm standby as the default, setting min_capacity=1 per region with scale-up at 70% CPU and scale-down to 1 (not 0). This saves $88/month versus always-on while eliminating cold start latency.

Both regions were deployed using Terraform v1.8.3 via workspaces prod-us-east-1 and prod-us-west-2. The us-east-1 health endpoint is live and returning 200. All changes were committed and pushed.
