The session covered a multi-threaded standup spanning infrastructure migration, authentication modernization, operational issues, and compliance work. The primary focus was migrating from EKS to GKE to achieve a 60% cost reduction (from $28,400/month to $11,200/month), while simultaneously cutting over authentication from static JWT validation to OIDC. The GKE cluster will be deployed as a single-zone instance initially to minimize cross-zone egress costs during validation.

A critical blocker emerged around persistent 504 errors on the dashboard metrics endpoint, affecting 0.7% of requests and exceeding the 0.5% SLA threshold. This issue is tracked in Sentry as PROJ-12847 and appears to be an upstream timeout to the analytics service. The team attempted a rollback to a previous build, but the issue persisted, indicating a pre-existing problem rather than a new regression. The dashboard workload migration to GKE is on hold until this is resolved, while stateless API workers will migrate first.

The OIDC cutover is proceeding with dual-path validation for 30 days to maintain backwards compatibility. The new OIDC issuer is live, and the auth middleware is being refactored to validate against it using the new key ID. A soft launch with read-only headers is planned for tomorrow, with hard cutover at week's end contingent on resolving the 504 issue. The rate limiter is ready with a Redis backend and tiered policies (free: 100/min, paid: 1000/min, enterprise: 10000/min), routing through a new API gateway.

Supporting work included database credential rotation with a 24-hour grace period, a VPC peering Terraform module awaiting network lead approval, onboarding a new SRE (Maya Lin) starting Monday with a two-week shadowing period, and SOC 2 Type II audit preparation for the Q2 2026 window. Evidence collection is underway with proper bucket access controls and encryption in place.
